就说一下碰到的一些东西吧
- waitForSelect、locator、$  这三个是不同的东西，waitForSelect一般是用来当作一个条件的比如等待那个元素出现之后再进行操作。locator是比较新的一种选择方法，这个我没太搞明白。$就是老式的写法
- locator 需要注意的是，locator不像$一样，locator只是定位到元素了，没有获取到元素，获取元素实在你对这个定位器对象进行操作的时候获取的。实际上拿到这个定位器的元素我也没有试过，好像是个handle啥的 这个支持多种选择器，下面这两种写法，第一张不知道是哪种写法，第二种是xpath，新版的puppeteer必须用这个语法 **::-p-xpath** 
```

await page.locator('input[placeholder="搜索作品"]').click();
await page.locator('::-p-xpath((//div[text()="用户"])[1])').click();
```

- $  万能的$符，这个好像是jquery留下来的，这里面只能写css选择器
- 选择器有xpath、css这两个选择器，可以说就是两种挑选元素的不同写法，这个需要了解。
- p站的网页上的class类名很多都是动态生成的，爬取的时候可以试试这样。1、如果元素有innertext的可以用xpath。2、如果是没有什么特殊的值比如类名也是动态的，但是有多重特殊的嵌套比如： **div > ul > li > div** 这种，就直接用css选择去选吧
- 这个流程太长了，每次重开太麻烦怎么办。可以试着打上 **debugger;** 然后用nodejs的调试模式一步步来。我看也有再浏览器里面直接开node的调试的，可以写一行代码运行一步。但是我的做法就是再当前的浏览器里面直接用选择器去选择一下看看能不能选到，选到了基本上就没有问题
  
## 下载
上面获取到图片的src了，现在只需要下载了。下载的思路就是向每个图片链接去发送fetch请求了，使用的是循环发送

```js
import fs from "node:fs"
import path from "node:path"
import { setGlobalDispatcher, ProxyAgent } from 'undici';




export const download = async (srcArr) => {

    if(!fs.existsSync('masheng')){
        fs.mkdirSync('masheng')
    }
    


    // const proxyUrl = 'http://127.0.0.1:33210';
    // // 配置Node.js的fetch使用代理
    // setGlobalDispatcher(new ProxyAgent(proxyUrl));
    const oriArr = srcArr.map((src) => {
        src = src.replace('_master1200', '');
        // src = strArr[0] + strArr[1];
        return src.replace('img-master', 'img-original')
    })

    console.log(oriArr);

    const dir = './masheng';
    for (let i = 0; i < oriArr.length; i++) {
        const url = oriArr[i];
        const res = await fetch(url, {
            headers: {
                'Referer': 'https://www.pixiv.net/',

            },
            cache: 'no-store'
        }) //这个是异步操作，因为要获取到res

        console.log('响应状态码：', res.status);

        const ex = res.headers.get('content-type').split('/')[1];

        let buffer = await res.arrayBuffer();  //这个也是流式的异步吗？不是，是promise需要await

        buffer = Buffer.from(buffer);
        const filepath = path.join(dir, `image-${i}.${ex}`)
        fs.writeFileSync(filepath, buffer)
    }

}
```
- 加上refers，不然服务器会觉得你是外来网站403，加上这个可以稍微告知是当前网站源的
- cache: 'no-store' 这个是禁用缓存的，有时候反复请求，它可能看到是同一个请求就返回给你缓存的图片，这个图片一般不是原图。状态码是304重定向
- fetch返回的res包含的body是这个图片的buffer，很长所以是一个类似流的传输，拿过来还需要转换一下再写入文件
- 字符串拼接操作，map、filter、split、splice、replace、join，需要熟悉一下了
- 代理，nodejs发送fetch请求的时候有网络的问题，可能是因为网站要挂梯子，后面使用proxy代理解决了，虽然不知道原理
