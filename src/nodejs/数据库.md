# 数据库

今天学了数据库，主要是数据库指令操作还有就是操作数据库
数据库：库里面有表，表里面有字段和数据，这两个相当于键值对。每个表都有主键，主键就是这个表的唯一标识
数据库操作：建库、建表、增删改查
建库建表不用说
对表的操作：
1. 给一个表插入一个字段，怎么写？
```
ALTER TABLE `user` ADD COLUMN `hobby` VARCHAR(100) COMMENT '爱好'
```
ALTER用来表示改变的操作嘛，再用COLUMN表示要加入的字段，COMMENT表示的是注释。git命令里的注释也是comment嘛

2. 删除一个字段 用DROP
3. 修改一个字段的信息，比如将age从int型改编成VARCHAR型的，用MODIFY
```
ALTER TABLE `user` MODIFY COLUMN `age` VARCHAR(100)
```


## 查询SELECT
1. 查询表中的所有数据
```
SELECT * FROM `user`;
```
这个*就是通配符，好像大部分地方都是这样写，这里可以查到表上所有的数据

2. 选择查询
查询某一列，比如查询id这一列。这里可以和模块化一样使用as给它换名字
SELECT id as user_id FROM `user`

3. **ORDER** 查询后按顺序排列
```
SELECT * FROM `user` ORDER BY id DESC; --降序
```
- DESC表示降序
```
SELECT * FROM `user` ORDER BY age ASC; --升序
```
- ASC 升序

4. **LIMIT** 分页查询
```
SELECT * FROM `user` LIMIT 0,3; 
```
- 查询前3条数据,从零开始，查三条 LIMIT

5. 条件查询
```
SELECT * FROM `user` WHERE name = 'hyx'; 
```
也是可以用AND和OR去把几个条件联合起来查询的


6. 模糊查询

```
SELECT * FROM `user` WHERE name LIKE 'h%';
```

- 模糊查询，查询名字中包含h的用户，这个和正则表达式中的%一样，表示任意字符，那这个就是以h开头的用户
```
SELECT * FROM `user` WHERE name LIKE '___h%';
```  

- 这里的下划线代表一个字符位

## 增删改查
1. 增加
```
INSERT INTO `user`(`name`,`age`,`hobby`) VALUES ('hyx', 20, 'reading'),('ymk',22,'有用');
```


2. 更改
```
UPDATE `user` SET name = 'hyx',age = 20, address = '九江' WHERE id = 8; --更新
```

3. 删除
```
DELETE FROM `user` WHERE id = 8; --删除  从哪里删除，然后接上条件
```

4. 查询
```
SELECT CONCAT(`name`,'无限') as name FROM `user`  --concat拼接
```

- 这样查出来的数据直接拼接了，前端不用再循环一遍

```
SELECT LEFT(`name`,2) as name FROM `user`
```

- left截取字符串左边的字符，括号内第一个参数是要截取的字段，第二个参数是要截取的长度

- 查询的时候还有一些操作：sum求和、avg平均值、count计数、now获取当前时间、date_add在原有的基础上添加日期

## 连表
### 连表操作
#### 内连接  两个表之间的关联关系

```SQL
SELECT * FROM `user`,`data` WHERE `user`.`id` = `data`.`user_id`
```

- user和user.id 对应 data和data.user_id对应

#### 外连接 -> 左连接 右连接
- 左连接 以左表为主，右表为辅，查询结果包含左
```
SELECT * FROM `user` LEFT JOIN `data` ON `user`.`id` = `data`.`user_id`
```

- 右连接 以右表为主，左表为辅，查询结果包含右

```SQL
SELECT * FROM `user` RIGHT JOIN `data` ON `user`.`id` = `data`.`user_id`
```

- 为主的这个表叫做**驱动表**

# knex

这是一个用js语句来写数据库操作的类似插件一样的东西，它会将按照它规范写的js语句转换成SQL然后执行

```node
npm istall knex --save
```

你使用的是什么数据库，就用什么，比如在这里使用mysql，就还是使用mysql2

## 实例化

```js
const db = knex({
    client: 'mysql2', //这里写数据库类型
    connection: config.db
})
```

## 建表

```js
db.schema.createTableIfNotExists('list', table => {
    table.increments('id'),  //主键
        table.string('name'),
        table.integer('age'),
        table.string('hobby'),
        table.timestamps(true, true)   //创建时间和更新时间
}).then(data => {
    console.log('表创建成功');

})
```

- 使用这个schema.createTableIfNotExists,意思也是一样的，不存在这个表就创建
- increments表示这个字段是主键，而且他这个id是会自增的
- 必须用then调用一下才会执行

## 事务
比如A，B两个人都在表里，现在进行交易，数据库有两个操作.1、A减100  2、B加100 。如果1语句结束，系统中断了，那么不是凭空少了100 ？
所一事务来解决这个情况，这种有依附关系的语句，把他们绑定在一起，要么一起完成，要么一起回退到任务开始

```js
app.get('/tran', (req, res) => {
    try{
      db.transaction(async trx => {
        await trx('caiwu').where({ id: 1 }).decrement('money',100);
        await trx('caiwu').where({ id: 2 }).increment('money',100);
        await trx.commit();//提交事务
    })
    res.send('OK')  
    }catch(err=>{
        await trx.rollback();//回滚事务
    })
    
})
```
